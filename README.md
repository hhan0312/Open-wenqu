# 问渠 Open Wenqu

AI 辅助命题、平行试题生成与质量评价工具。

问渠是一个面向教师和教研场景的开源项目，英文名为 Open Wenqu。它的目标不是做一个“AI 自动出题神器”，而是做一个可审核、可解释、可编辑的 AI 辅助命题工具，帮助教师围绕课程内容和考试目标生成平行试题，并自动完成答案、解析、证据、干扰项分析和质量检查。

“问渠”取自“问渠那得清如许，为有源头活水来”。这个名字强调两件事：一是从文本源头寻找答案依据，二是通过不断追问形成更高质量的题目。

## 项目定位

第一阶段先聚焦一个清晰场景：

> 高中英语 -> 阅读理解 -> 细节理解题 / 推理判断题 -> 平行试题生成与质量评价

教师可以粘贴一篇阅读材料和原题，系统分析原题考点，再生成同题型、同难度、同能力要求的平行题。生成结果由教师审核、编辑后再导出使用。

本项目强调：

- 辅助教师，而不是替代教师
- 生成平行题，而不是随机出题
- 结构化输出，而不是只给一段自然语言答案
- 质量评价可解释，而不是只给一个模糊评分
- 教师可编辑、可审核、可导出

## 为什么先做高中英语阅读理解

高中英语题型很多，包括听力、阅读、七选五、完形填空、语法填空、应用文写作、读后续写等。如果一开始做全学科、全题型，产品边界会过大，质量很难控制。

阅读理解中的细节理解题和推理判断题更适合作为第一阶段：

- 阅读材料和答案证据都在文本中，便于校验
- 题目生成结果可以结构化表示
- 正确答案、证据句、干扰项可以被解释
- 教师能快速判断生成质量
- 适合后续扩展到主旨题、词义猜测题、七选五等题型

## MVP 目标

第一版只完成一个核心闭环：

1. 教师粘贴阅读材料
2. 选择题型、年级、难度和生成数量
3. 系统生成阅读理解四选一题目
4. 自动给出答案、证据句、中文解析和干扰项分析
5. 系统进行质量评价
6. 教师编辑题目
7. 导出学生版和教师版

第一版优先支持：

- 阅读理解细节理解题
- 阅读理解推理判断题

## 核心功能

### 1. 原题分析器

输入阅读材料和原题，自动分析：

- 题型：细节理解、推理判断、主旨大意、词义猜测等
- 考点：信息定位、同义转述、因果推断、观点态度等
- 答案证据：原文中的段落和句子
- 正确选项依据：为什么正确
- 干扰项类型：为什么错误

### 2. 平行试题生成器

在保持核心考查目标不变的前提下生成新题。

生成时保持不变：

- 考查能力不变
- 难度层级不变
- 题型结构不变
- 选项数量不变
- 答案唯一性不变

允许变化：

- 语境
- 人物
- 材料内容
- 设问角度
- 选项表达

### 3. 干扰项生成器

阅读理解题的质量很大程度取决于干扰项。本项目会对干扰项进行分类生成和分析。

常见干扰项类型：

- 无中生有
- 偷换概念
- 张冠李戴
- 范围扩大
- 范围缩小
- 因果倒置
- 正误混杂
- 常识正确但原文无依据

### 4. 质量评价器

每道题生成后，系统自动检查：

- 答案是否唯一
- 证据是否充分
- 干扰项是否合理
- 题干是否清晰
- 难度是否匹配目标年级
- 词汇是否适配高中阶段
- 与原题是否过于相似
- 输出格式是否完整

示例评价：

```text
质量评分：86/100

优点：
- 答案证据明确，位于第 2 段第 3 句。
- B、D 选项具有一定干扰性。

问题：
- A 选项明显错误，干扰性较弱。
- 题干中的 according to the passage 可改为 according to paragraph 2，更利于定位。
```

## 输出格式

每道题使用结构化 JSON 表示，便于后续校验、保存、编辑和导出。

```json
{
  "passage_id": "reading_001",
  "question_type": "detail",
  "stem": "What does the author say about ...?",
  "options": {
    "A": "...",
    "B": "...",
    "C": "...",
    "D": "..."
  },
  "answer": "C",
  "evidence_sentence": "...",
  "explanation_cn": "...",
  "distractor_analysis": {
    "A": "无中生有",
    "B": "偷换概念",
    "D": "范围扩大"
  },
  "difficulty": "medium",
  "tested_skill": "定位具体信息并进行同义转述",
  "quality_score": 86
}
```

## 推荐技术栈

第一版以完整闭环为优先，不追求复杂架构。

- 前端：Next.js 或 Vue
- 后端：FastAPI
- 数据库：PostgreSQL
- 向量检索：pgvector
- 文件解析：docx、pdf、markdown
- 模型调用：OpenAI API、Claude、通义、DeepSeek 或本地模型
- 导出格式：Markdown、Word、PDF

## 系统流程

## 数据库设计草案

第一版至少需要以下数据表：

- users：用户表
- passages：阅读材料表
- questions：题目表
- generation_tasks：生成任务表
- question_reviews：教师评价表
- prompt_templates：提示词模板表

题目表示例：

```sql
CREATE TABLE questions (
  id SERIAL PRIMARY KEY,
  passage_id INTEGER,
  subject VARCHAR(50),
  grade VARCHAR(20),
  question_type VARCHAR(50),
  stem TEXT,
  options JSONB,
  answer VARCHAR(10),
  explanation TEXT,
  evidence TEXT,
  distractor_analysis JSONB,
  difficulty VARCHAR(20),
  quality_score NUMERIC,
  created_at TIMESTAMP DEFAULT NOW()
);
```

## 第一版提示词模板

```text
你是一名高中英语命题教师。请根据给定阅读材料，生成 {数量} 道高中英语阅读理解细节理解题。

要求：
1. 题型为四选一选择题。
2. 每题只有一个最佳答案。
3. 正确答案必须能在原文中找到明确证据。
4. 干扰项必须合理，不能明显错误。
5. 题干应符合高考英语阅读理解命题风格。
6. 选项长度应基本一致。
7. 请提供答案、证据句、中文解析和干扰项分析。
8. 输出 JSON 格式。

阅读材料：
{passage}

目标年级：
{grade}

难度：
{difficulty}
```

## 开发路线图

### 第 1 周：原型

- 粘贴阅读材料
- 选择题型
- 调用大模型生成题目
- 返回题干、选项、答案和解析

目标：能用。

### 第 2 周：结构化输出

- 设计 JSON Schema
- 做题目格式校验
- 做答案字段校验
- 做证据句字段校验
- 支持 Markdown 导出

目标：输出规范。

### 第 3 周：质量评价

- 答案唯一性检查
- 证据句检查
- 干扰项类型分析
- 难度评分
- 相似度检测

目标：老师敢用。

### 第 4 周：教师编辑和导出

- 在线编辑题干和选项
- 重新生成单个选项
- 保存题目
- 导出学生版
- 导出教师版

目标：可以给真实教师试用。

## 后续扩展方向

推荐扩展顺序：

1. 阅读理解细节理解题
2. 阅读理解推理判断题
3. 阅读理解主旨大意题、词义猜测题
4. 七选五
5. 完形填空
6. 语法填空
7. 应用文写作
8. 读后续写
9. 听力材料与听力题生成

## 项目愿景

本项目希望成为一个开源的英语阅读命题辅助工具，帮助教师更高效地完成平行试题生成、干扰项设计和试题质量检查。

它不是为了让 AI 完全接管命题，而是为了把教师最耗时、最重复、最需要初稿辅助的环节产品化，让教师把更多精力放在审核、修改、教学判断和学生反馈上。
